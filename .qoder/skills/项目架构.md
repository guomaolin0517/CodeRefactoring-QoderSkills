## 1. 项目概述

本项目是一个基于 **Spring Cloud** 的微服务架构系统，主要用于**财会监督平台**的管理和运营。系统采用**模块化设计**，将功能划分为多个独立的服务和组件，实现了**高内聚、低耦合**的架构目标。

## 3. 项目整体结构

体验层：Controller模块，负责处理HTTP请求和响应，与前端交互+feign接口封装。

聚合层：承上启下，衔接体验层与能力层，核心做「数据聚合、接口调用、数据组装」

能力层：提供的能力、功能、独立，单独的文件夹(/service/common)，接口和实现类

底座层：平台的依赖、基础设施(redis/Ai等)、数据源配置等

## 7. 核心流程

### 7.1 配置管理流程

```
┌─────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  用户           │───►│  ufsp-web   │───►│ ufsp-config │───►│  配置数据库        │
│         │    │             │    │             │    │             │
│ 操作配置    │◄───│   转发请求          │◄───│  处理配置请求   │◄───│  更新/查询          │
└─────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

**流程说明**：
1. 用户通过前端界面发起配置操作
2. `ufsp-web` 接收并转发请求到 `ufsp-config`
3. `ufsp-config` 处理配置请求，调用相应的配置管理组件
4. 配置数据更新或查询数据库
5. 返回配置结果给前端

             │  问题处理             │       │  tion     │       │   模块              │

```
┌─────────────┐    ┌─────────┐    ┌─────────────┐    ┌─────────────┐

**流程说明**：
1. `ufsp-etl` 定期或触发式执行数据同步任务

4. 将处理后的数据同步到目标系统

## 10. 总结

### 10.1 架构特点

本项目采用**微服务架构设计**，具有以下特点：

| 特点 | 说明 |
|:---|:---|
| **模块化设计** | 功能划分为多个独立模块，高内聚低耦合 |
| **层次分明** | 接入层 → 服务层 → 业务层 → 支撑层 → 数据层 |
| **灵活扩展** | 支持水平扩展，可根据负载动态调整 |
| **多环境支持** | 支持 Spring Cloud 和 TSF-GD 两种部署模式 |
| **国产适配** | 支持多种国产数据库（人大金仓、达梦） |

### 10.2 核心优势

1. **🎯 业务聚焦**：以财会监督闭环管理为核心，所有模块围绕核心开展工作
2. **🔧 技术先进**：采用 Spring Cloud 微服务架构，支持容器化部署
3. **📊 数据驱动**：ETL 数据同步 + 模型分析，支持数据驱动决策
4. **🛡️ 安全可靠**：多数据源支持、分布式事务、服务熔断降级
5. **🚀 易于维护**：模块化设计，各模块独立开发、独立部署
